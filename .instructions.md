# DubMux Agent Instructions

Этот файл содержит инструкции для агентов при разработке и поддержке проекта DubMux.

## Обзор проекта

**DubMux** — десктоп-приложение для автоматического подбора и смешивания внешних аудиофайлов с эпизодами сериалов/мультиков.

- **Фреймворк**: Tauri 2.0 (Rust + React)
- **Фронтенд**: React 18.3.1 + TypeScript 5.5.4 + Vite 5.4.21
- **Бэкенд**: Rust (Tauri commands)
- **Инструменты**: FFmpeg/FFprobe для мультиплексирования
- **Платформа**: Windows (NSIS installer)
- **Версия**: 0.1.x

## Ключевые компоненты

### Фронтенд: `src/App.tsx`

**Состояние (State):**

- `videoFolder`, `audioFolder`, `outputFolder` — пути к папкам
- `matchPreview` — результат автоподбора
- `manualMode` — режим ручного подбора
- `manualMatches`, `audioOverrides` — ручные переопределения
- `isProcessing`, `processingProgress` — статус обработки
- `processingSuccessMessage`, `processingErrors` — результаты

**Основные функции:**

- `pickFolder(label)` — нативный диалог выбора папки
- `startMatching()` — запуск алгоритма подбора
- `startProcessing()` — запуск FFmpeg мультиплексирования
- `handleMuxProgress()` — обработка событий прогресса

### Matching Engine: `src/features/matching/matching.ts`

**Основная функция: `buildMatchPreview()`**

Алгоритм:

1. Сканирует видео- и аудио-папки
2. Классифицирует файлы по расширениям (`.mkv`, `.mp4`, `.aaa`, `.m4a` и т.д.)
3. Извлекает номер эпизода из имени файла (6 встроенных паттернов):
   - `S01E03` (стандартный формат)
   - `1x03` (альтернативный формат)
   - `Episode 03`
   - Числовые токены (например, `01` из `ep01.mkv`)
   - Пользовательский regex (захватывающая группа 1)
   - Фикс: отрицательное lookaround для исключения разрешений (1440x1080)

4. Группирует видео и аудио по номеру эпизода
5. Возвращает `MatchPreview[]` с указанием конфликтов

**Важно:**

- Нормализация номеров: `103` → `3` (если есть неоднозначность)
- Regex обрабатывается в `extractEpisodeNumber()`
- Поддерживаемые форматы: `.mkv`, `.mp4`, `.avi`, `.aac`, `.m4a`, `.mp3`, `.flac`, `.wav`, `.mka`

### Job Planning: `src/features/processing/jobPlan.ts`

**Функции:**

- `validateJobPlan(preview, mode)` — валидация перед обработкой
  - Auto mode: все видео должны быть подобраны
  - Manual mode: требует явного подбора каждого файла
- `buildJobPlan()` — создание массива `MuxJob` для автоматического подбора
- `buildManualJobPlan()` — для ручного подбора
- `estimateProcessingTime()` — приблизительная оценка (2–5 мин/файл)

### FFmpeg Backend: `src-tauri/src/commands/mux.rs`

**Основная команда: `process_mux_jobs()`**

Параметры:

```rust
jobs: Vec<MuxJob>,
app_handle: AppHandle,
```

Логика:

1. Обходит каждый job последовательно
2. Для каждого файла:
   - Вызывает `count_audio_streams()` (ffprobe) — считает существующие аудиодорожки
   - Вызывает `mux_single_file()` — выполняет ffmpeg
   - Испускает событие `mux_progress` (текущий файл, статус)
3. Обработка ошибок: логирует путь к бинарнику, exit code, stderr/stdout

**FFmpeg команда:**

```bash
ffmpeg -i video.mkv -i audio.aac \
  -map 0 -map 1:a:0 \
  -c copy -c:a:N aac \
  -metadata:s:a:N title=<audio_track_name> \
  -shortest -y output.mkv
```

**Разрешение пути к бинарнику (`resolve_binary_path()`):**

1. `resource_dir/bin/ffmpeg.exe` (bundled)
2. Директория exe файла (`app.exe` директория)
3. Системный PATH (`which ffmpeg`)

Вывод ошибок включает попытанные пути для отладки.

### Tauri Configuration: `src-tauri/tauri.conf.json`

Важные секции:

- `bundle.resources`: включает `["bin/ffmpeg.exe", "bin/ffprobe.exe"]`
- `package.version`: версия приложения (синхронизировать с `package.json`)
- `productName`: "DubMux"

## Release Pipeline (GitHub Actions)

Файл: `.github/workflows/release.yml`

**Триггер:** Push tag `v*` или ручной `workflow_dispatch`

**Шаги:**

1. Checkout кода
2. Setup Node.js 20 + yarn cache
3. Setup Rust toolchain
4. `yarn install`
5. Скачивание FFmpeg с `gyan.dev` → `src-tauri/bin/`
6. Запуск `tauri-apps/tauri-action@v0` для сборки NSIS
7. Публикация GitHub Release с `.exe` приложением

**Важно:**

- FFmpeg скачивается как portable .zip из gyan.dev (не через Chocolatey, чтобы избежать shim-проблем)
- Бинарники распаковываются в `src-tauri/bin/` перед сборкой

## Известные проблемы и решения

### Проблема: FFmpeg не найден в релизе

**Причина:** Chocolatey создаёт wrapper scripts вместо реальных исполняемых файлов

**Решение:** GitHub Actions теперь скачивает portable FFmpeg с gyan.dev

**Проверка:** Убедитесь, что в release.yml есть PowerShell скрипт для распаковки

### Проблема: UI зависает во время обработки

**Причина:** FFmpeg работает в основном потоке

**Решение:** `spawn_blocking()` для вызова `mux_single_file()`

### Проблема: Внешний аудиофайл заменяет оригинальный звук

**Причина:** Неправильная FFmpeg маппинга (`-map 1:a:0` вместо `-map 0`)

**Решение:** Использовать `-map 0 -map 1:a:0` (добавить, не заменять)

## Рекомендации для агентов

### При добавлении функций

1. **Новый regex паттерн**: добавить в `extractEpisodeNumber()` и написать тест в `matching.test.ts`
2. **Новая команда Tauri**:
   - Реализовать в `src-tauri/src/commands/`
   - Зарегистрировать в `src-tauri/src/lib.rs`
   - Добавить TypeScript bindings в `src/features/mux.ts`
3. **UI изменения**: обновить `App.tsx` и `styles.css`
4. **FFmpeg команда**: проверить с реальными видео/аудио файлами перед запуском

### При отладке

- Проверить консоль браузера (F12 в dev режиме) — там Tauri логи
- Проверить `src-tauri/src/commands/mux.rs` — логирование binary path и exit codes
- Для VM: убедитесь, что WebView2 установлен и есть сетевой доступ
